# Game Pillars

这份文件用来固定《BING / 饼》必须带给玩家的核心感受。任何新功能、UI 改动和美术资产，如果不服务这些支柱，就应该重新评估优先级。

## 1. Core Feeling

玩家应该感到：

- 紧张：所有玩家同时决策，亮招前有明确的心理博弈。
- 掌控：资源、目标、阶段和提交状态必须一眼可读。
- 爽快：亮招、命中、防御、反弹、技能触发要有清楚反馈。
- 深渊感：牌桌像一台遗物装置，角色像被卷入深层竞技场。
- 可复盘：事件日志、回放和 UI 状态能解释“为什么刚才发生了这个结果”。

## 2. Core Loop

当前核心循环：

```text
观察桌面 -> 选择行动 -> 同时提交 -> 亮招结算 -> 获得资源/伤害/信息 -> 进入下一次博弈
```

每一轮都应该让玩家回答三个问题：

- 我现在能做什么？
- 我猜对手会做什么？
- 如果猜错，我还能用什么技能或防御补救？

## 3. Player Promise

玩家看到 Steam 页面或 README 时，应该一眼明白：

```text
这是一款同时行动制卡牌策略游戏。
它最吸引人的点是“所有人暗中出招，然后一起亮牌结算”的心理博弈。
玩家会反复想玩的原因是：每一回合都短、狠、可复盘，并且技能组合会制造新的猜法。
```

## 4. Design Pillars

- 同时行动的读秒压力：倒计时、提交状态和未提交名单永远清楚。
- 低门槛的第一局：新手能创建房间、加入房间、吃饼、提交，不需要外部说明。
- 可读的复杂性：技能可以复杂，但 UI 必须把当前阶段、目标、消耗和结果翻译成人话。
- 游戏质感优先：界面像游戏桌面/HUD，不像普通网页后台。

## 5. Non-goals

明确不做的事：

- 不做大规模开放世界或剧情探索。
- 不做与核心出招循环无关的养成系统。
- 不做大量装饰性 UI 动效，所有动效都要帮助玩家理解状态变化。
- 不为了“看起来复杂”堆信息，当前决策用不到的信息要收进 tabs、日志或规则面板。

## 6. Current North Star

当前最重要的阶段目标：

```text
做出一段 5-10 分钟、能代表最终品质的多人垂直切片：
从开房、加入、选择角色、开始游戏，到连续 3 回合出招、亮招、看懂结算，并能生成 playtest 报告。
```
