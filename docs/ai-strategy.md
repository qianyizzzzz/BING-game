# 《饼》AI 策略说明

本文档记录当前 AI 玩家如何出招、如何使用训练权重，以及后续继续训练时应该重点观察什么。

## 1. 第一回合规则

当前代码强制 AI 在总第 1 回合出饼：

```ts
if (state.config.firstTurnNoAttack && state.turnNumber === 1) {
  return { type: "gain_cake" };
}
```

原因是基础规则明确写着“第一回合禁止攻击，只能出饼或防御”。AI 如果第一回合不出饼，会浪费最稳定的发育窗口，所以这里不走训练权重，直接固定为出饼。

## 2. 候选动作

每次轮到 AI 出招时，它会枚举所有合法动作：

- 出饼。
- 小防、油条、石头。
- 有饼时可以反弹，默认弹向血量最低的敌人。
- 饼足够时枚举基础攻击，最多考虑 3 重。
- 如果拥有可主动施放的技能，也会枚举技能动作，例如圣、钟、梵、连爆等。

第一回合不会枚举攻击和技能。

## 3. 基础评分

AI 会给每个候选动作一个基础分：

- 出饼：前期、低饼量、桌面压力低时更高。
- 防御：低血量或敌人有饼时更高。
- 反弹：低血量、敌人有饼、自己饼多时更高。
- 攻击：伤害、等级、费效比、斩杀可能、群攻收益都会提高分数。
- 技能：攻击型技能按攻击逻辑评分，资源型技能按获得饼量和当前压力评分。

最后从得分最高的前 3 个动作里按权重随机选择，避免每局完全机械重复。

## 4. 自我对战训练权重

训练脚本是：

```bash
npm.cmd run training:selfplay -- --duration-minutes 120 --skill-heavy --checkpoint-games 1000
```

训练后会写入：

- `data/ai/ai-policy.json`
- `data/ai/self-play-report.txt`

服务器 AI 每次评分时会读取 `ai-policy.json`，把动作特征对应的训练权重加到基础分里。正权重表示这个特征在自我对战里更常和胜利相关，负权重表示更常和失败相关。

## 5. 技能局策略

当前训练会偏向带技能局：

- 约 90% 的训练局带 1 到 3 张小技能。
- 资源技能会提高 AI 的中期爆发。
- 攻击型技能会进入和基础攻击相同的目标选择逻辑。
- 锁定技会自动通过结算引擎改变基础攻击，例如火焰刀和朱雀羽扇。

## 6. 当前限制

- 复杂控制技、限定技、时机技还没有全部接入完整时机系统。
- AI 现在主要学习“动作特征和胜负的线性关系”，还不是深度强化学习。
- 多人局里的联盟、诱骗、保留反弹等高级心理博弈还需要更多真实玩家数据。

## 7. 后续训练建议

- 多导出真人对局样本，让 AI 不只学习自我对战。
- 把技能按“攻击、资源、防御、控制、限定、锁定”继续拆出明确的结算钩子。
- 给每次训练记录胜率、平均回合数、技能使用率、反弹收益，避免 AI 只学会盲目进攻。
