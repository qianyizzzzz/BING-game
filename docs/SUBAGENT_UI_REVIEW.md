# BING 子智能体 UI 评审汇总

日期：2026-06-12

本轮评审使用的是 Codex 侧边栏里的运行态子智能体，而不是 `scripts/ui-playtest-agents.ts` 里的 Playwright 测试 persona。Playwright 脚本仍然保留，用于后续自动化截图、流程回归和报告生成。

当前 repo 尚未包含 `.agents` 持久化配置；如果要让这些角色在项目里长期固定出现，需要下一步补充对应的 agent 配置文件。

## 参与角色

- 新手玩家 Agent：第一次玩 BING / 饼，关注能不能创建房间、加入、开始、提交第一回合。
- 竞技玩家 Agent：高频对局用户，关注重复出招速度、目标信息、资源读取、动画是否干扰判断。
- 开发商 Agent：制作人 / QA / 技术负责人，关注计划完成度、测试闭环、性能、可访问性和发布风险。
- 美术总监 Agent：关注 art direction、角色资产、技能视觉、动效节奏和真实游戏质感。

## 总结判断

项目已经从“网页式卡牌工具”明显走向“深渊遗物牌桌竞技”的方向。首屏、战斗 HUD、行动 dock、资源变化、事件 timeline mapper、中文 README 都已经有基础。

主要缺口是：

- 新手路径需要更明确的下一步提示。
- 竞技玩家缺少快速重复出招、目标预览、未提交名单。
- 战斗动画仍偏全屏结算弹层，尚未完全变成桌面内演出。
- 美术语言仍混有 placeholder、英文 sigil、网页 modal 感。
- 自动化测试 persona 与真实 Codex 子智能体需要分工清楚。

## 已根据评审立即处理

- 大厅增加下一步提示：房主能看到“加 AI / 分享房号 / 开始”的下一步。
- `开始` 禁用时显示原因：人数不足、等待房主、忙碌处理会有不同提示。
- 非房主不再看到可误点的 `加 AI / 开始` 主操作，而是显示“等待房主开始”。

对应 commit：`Clarify lobby start guidance`。

## P0 修改意见

1. 完成公网试玩前验证闭环。

   验收：`npm.cmd run build`、本地 `serve`、`test:ui-agents`、桌面和 390px 移动端截图报告都能产出。

2. 保证新手第一局不会卡住。

   验收：单人房间时能看到还差几名玩家；非房主知道要等房主；第一手“吃饼 +1”的含义清楚。

3. 保留核心牌桌信息，减少结算遮挡。

   验收：动画播放时仍能看到玩家 HP / 饼 / 已出招状态，或提供精简结算条。

## P1 修改意见

1. 目标预览。

   选择攻击或技能目标后，目标座位出现高亮或桌面连线；切换目标实时更新。

2. 重复上回合。

   行动面板增加“沿用上回合”按钮。合法时一键提交；非法时说明原因。

3. 未提交名单。

   HUD 不只显示 `x/y 已出招`，还显示还差哪些玩家。

4. 上下文右栏。

   战斗中默认显示 `提示 / 日志 / 技能 / 规则` tabs，减少 EventLog、SkillPanel、TutorialPanel、ReferencePanel 同屏抢注意力。

5. 事件日志和动画口径对齐。

   `turnTimeline` 已映射 `skill_used` 等事件；EventLog 的过滤策略也应能让 QA 追踪“哪些事件有视觉反馈”。

6. 强化 `scripts/ui-playtest-agents.ts`。

   给关键 UI 加 `data-testid`，脚本检查 console error、截图、元素重叠、canvas 非空、行动提交后状态变化。

## P2 修改意见

1. 拆分 `ActionPanel.tsx`。

   建议拆成 `MainActionForm`、`ActionWindowPanel`、`SkillTargetControls`、`ActionSummaryDock`，降低后续技能和快捷键维护成本。

2. 补完整 Art Bible。

   定义色板、材质、按钮形状、字体层级、角色比例、技能符号、动效节奏。

3. 音效注册表。

   基于 `BattleSoundCue` 先做空实现，再接入点击、提交、亮招、命中、防御、反弹、回血、死亡、胜利音效。

4. 视觉回归。

   Playwright 固定桌面和移动端截图，检查首屏、牌桌、行动 dock、结算动画、reduced motion。

## 动效设计方案

目标：把回合结算从“弹层播报”推进到“桌面内游戏演出”。

固定节拍：

1. 锁定：已出招座位轻微脉冲，未出招座位保持等待态。
2. 亮招：行动卡从座位翻到桌心。
3. 预备：镜头轻推，目标线出现，桌面光圈收缩。
4. 命中：80-140ms hit-stop，目标抖动，粒子爆发，触发 sound cue。
5. 结算：HP / 饼数字滚动变化，死亡、反弹、回血单独演出。
6. 收束：镜头回中，HUD 提示下一步。

建议给 `BattleStep` 继续扩展：

- `hitStopMs`
- `cameraCue`
- `shakeIntensity`
- `particlePreset`
- `importance`

## 美术方案

主方向：深渊遗物牌桌竞技。

需要统一：

- 移除 meadow / placeholder 背景与深渊牌桌的冲突。
- 结算层减少白底网页 modal 感，改为桌面内暗色浮层或桌心演出。
- 技能 sigil 从英文 `Relic / PYR / FRS` 改成中文或图形符号，例如 `焰 / 霜 / 雷 / 蚀 / 盾 / 命 / 虚 / 遗`。
- 统一 2D 座位头像和 3D 桌面角色语言。

资产清单：

- 6 张角色头像，512x512。
- 6 个座位半身或小人形象。
- 8 套技能派系 icon。
- 吃饼、防御、攻击、技能 4 个行动 icon。
- 卡背、弃牌堆、桌面纹理。
- 攻击火花、防御盾环、反弹弧线、破弹裂纹、回血粒子、死亡暗化粒子。
- UI 点击、提交、亮招、命中、防御、反弹、回血、死亡、胜利音效。

## 子智能体工作流

每次重要 UI commit 后建议执行：

1. 新手玩家 Agent：检查第一局路径和文案。
2. 竞技玩家 Agent：检查高频操作效率和读局速度。
3. 开发商 Agent：检查 bug、测试、性能、可访问性、发布风险。
4. 美术总监 Agent：检查风格统一、资产质量、动效节奏和截图审美。
5. 主线程汇总意见，生成 `artifacts/playtests/` 报告或更新本文件。

Playwright `test:ui-agents` 用于机器验证；Codex 子智能体用于主观体验、设计判断和修改建议。
